Desenvolvendo com Ethereum (dApp)

O que é uma Aplicação Descentralizada (dApp)?

Uma aplicação descentralizada (dApp) é um tipo de software que existe e roda em uma blockchain usando comunicação ponto a ponto. O “backend” roda em muitos “nós” distribuídos em vez de um único servidor centralizado.

Arquitetura Web2 Tradicional vs Web3

Web2 (Internet Atual)

  • Servidores centralizados
  • Dados pertencem às plataformas
  • Usuários são o produto
  • Pontos únicos de falha
  • Possibilidade de censura
  • APIs controladas por empresas

Exemplos: Facebook, Google, Amazon

Web3 (Web Descentralizada)

  • Redes distribuídas
  • Usuários possuem seus dados
  • Dinheiro programável
  • Imutável e transparente
  • Resistência à censura
  • Interação ponto a ponto

Exemplos: Uniswap, OpenSea, Compound

Qualquer “dApp” precisa de uma forma de se comunicar com a blockchain!

Estrutura e Componentes de uma dApp

DAPP_Equação = I + II

Dois Lados da Equação

I: Desenvolvimento de Smart Contracts

  • Lógica de backend implantada na blockchain
  • Escritos em Solidity/Vyper
  • Imutáveis após o deploy
  • Contratos autoexecutáveis
  • Sem intermediários

II: Aplicação Cliente

  • Interface frontend (HTML/CSS/JavaScript)
  • Interage com a blockchain
  • Interface amigável ao usuário
  • Integração com carteiras
  • Tratamento de transações

JSON-RPC e Provedores Web3

O que é JSON-RPC?

JSON-RPC (JavaScript Object Notation - Remote Procedure Call) é um protocolo de chamada de procedimento remoto, sem estado e leve, codificado em JSON, que permite a comunicação entre aplicações cliente e nós Ethereum.

Detalhes do Protocolo JSON-RPC

Fluxo de Comunicação:

Linguagem de Programação → JSON-RPC (Habilitado por Web3.js) → Cliente Ethereum → Blockchain

Métodos JSON-RPC Comuns

eth_getBalance

Obtém o saldo da conta em Wei

eth_sendTransaction

Envia transações para a rede

eth_call

Chama funções de smart contracts (somente leitura)

eth_getBlock

Obtém informações e dados de blocos

Características Principais

  • Protocolo sem Estado: Cada requisição é independente e autônoma
  • Agnóstico à Linguagem: Funciona com JavaScript, Python e outras linguagens
  • Padronizado: Nós Ethereum expõem métodos de API consistentes
  • Flexível no Transporte: Funciona via HTTP, WebSocket e IPC

Sistema de Provedores do Web3.js

Tipos de Provedores no Web3.js

HttpProvider

Conexão: HTTP para nós remotos (Infura, Alchemy)

  • Melhor para operações de leitura
  • Configuração simples
  • Amplo suporte entre plataformas
  • Maior latência que WebSocket

WebSocketProvider

Conexão: Em tempo real com assinaturas de eventos

  • Escuta de eventos em tempo real
  • Operações com menor latência
  • Conexão persistente
  • Perfeito para dApps ao vivo

IpcProvider

Conexão: IPC local para instâncias de nó

  • Comunicação local direta
  • Método de conexão mais rápido
  • Requer nó da blockchain local
  • Ideal para desenvolvimento

Provedores Personalizados

Conexão: Extensível para necessidades especializadas

  • Personalizável para redes específicas
  • Integração com infraestrutura proprietária
  • Opções avançadas de configuração
  • Soluções corporativas

Vantagens da Arquitetura Web3.js

  • Biblioteca Madura: Uma das primeiras e mais estabelecidas bibliotecas Ethereum
  • API Abrangente: Cobertura total dos métodos JSON-RPC do Ethereum
  • Grande Comunidade: Muitos tutoriais, documentação e suporte
  • Pronto para Produção: Usada por grandes dApps com confiabilidade comprovada
  • Abordagem de Instância Única: Uma instância web3 lida com operações de leitura e escrita
  • Utilitários Embutidos: Funções auxiliares abrangentes para operações comuns
