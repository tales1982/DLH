Developing with Ethereum (dApp)

What is a Decentralized Application (dApp)?

A decentralized application (dApp) is a type of software that exists and runs on a blockchain using peer-to-peer communication. The "backend" runs across many distributed "nodes" instead of a single centralized server.

Traditional Web2 vs Web3 Architecture

Web2 (Current Internet)

  • Centralized servers
  • Data owned by platforms
  • Users are products
  • Single points of failure
  • Censorship possible
  • APIs controlled by companies

Examples: Facebook, Google, Amazon

Web3 (Decentralized Web)

  • Distributed networks
  • Users own their data
  • Programmable money
  • Immutable & transparent
  • Censorship resistance
  • Peer-to-peer interaction

Examples: Uniswap, OpenSea, Compound

Any "dApp" needs a way to communicate with blockchain!

dApp Structure and Components

DAPP_Equation = I + II

Two Sides of the Equation

I: Smart Contract Development

  • Backend logic deployed on blockchain
  • Written in Solidity/Vyper
  • Immutable once deployed
  • Self-executing contracts
  • No intermediaries needed

II: Client Application

  • Frontend interface (HTML/CSS/JavaScript)
  • Interacts with blockchain
  • User-friendly interface
  • Wallet integration
  • Transaction handling

JSON-RPC and Web3 Providers

What is JSON-RPC?

JSON-RPC (JavaScript Object Notation - Remote Procedure Call) is a stateless, lightweight remote procedure call protocol encoded in JSON that enables communication between client applications and Ethereum nodes.

JSON-RPC Protocol Details

Communication Flow:

Programming Language → JSON-RPC (Enabled by Web3.js) → Ethereum Client → Blockchain

Common JSON-RPC Methods

eth_getBalance

Retrieve account balance in Wei

eth_sendTransaction

Send transactions to the network

eth_call

Call smart contract functions (read-only)

eth_getBlock

Retrieve block information and data

Key Characteristics

  • Stateless Protocol: Each request is independent and self-contained
  • Language Agnostic: Works with JavaScript, Python, and other languages
  • Standardized: Ethereum nodes expose consistent API methods
  • Transport Flexible: Works over HTTP, WebSocket, and IPC

Web3.js Provider System

Provider Types in Web3.js

HttpProvider

Connection: HTTP to remote nodes (Infura, Alchemy)

  • Best for read operations
  • Simple setup and configuration
  • Wide support across platforms
  • Higher latency than WebSocket

WebSocketProvider

Connection: Real-time with event subscriptions

  • Real-time event listening
  • Lower latency operations
  • Persistent connection
  • Perfect for live dApps

IpcProvider

Connection: Local IPC to node instances

  • Direct local communication
  • Fastest connection method
  • Requires local blockchain node
  • Ideal for development

Custom Providers

Connection: Extensible for specialized needs

  • Customizable for specific networks
  • Integration with proprietary infrastructure
  • Advanced configuration options
  • Enterprise solutions

Web3.js Architecture Advantages

  • Mature Library: One of the first and most established Ethereum libraries
  • Comprehensive API: Full coverage of Ethereum JSON-RPC methods
  • Large Community: Extensive tutorials, documentation, and support
  • Production Ready: Used by major dApps with proven reliability
  • Single Instance Approach: One web3 instance handles both read and write operations
  • Built-in Utilities: Comprehensive helper functions for common operations
